# Issue #99 deviations and inventions

## Invented names

- `HypsterRenderer`, `HypsterRendererProps`, `InteractiveSnapshot`, and `InteractiveAction`: the public renderer seam and Protocol V1 TypeScript names requested by the ticket.
- `InteractiveParameter`, `InteractiveParameterKind`, `InteractiveScalar`, `InteractiveValue`, `InteractiveValues`, `InteractiveSchema`, `InteractiveStatus`, `InteractiveError`, `FieldSpec`, `RulesMetadata`, `SchemaField`, and `SchemaMetadata`: TypeScript names for the existing snake_case Python snapshot data.
- `ParameterBase`, `ParameterField`, `ParameterControl`, `JsonControl`, `encoded`, and `snapshotValue`: private presentation types/components/functions that render snapshot data without owning configuration state.
- `.github/workflows/react.yml`: dedicated path-filtered React package CI workflow.
- `test-build-pack`: the React CI job name.
- `packages/react/test/renderer.test.mjs` plus its private `parameter`, `snapshot`, and `renderer` factories: package-level jsdom acceptance suite.

## Granted guesses

- Protocol V1 snapshots are typed with required `protocol_version: 1` even though runtime mismatch suppression and Python-generated fixtures belong to #100.
- Multi-value, rules, and schema values use JSON text controls. JSON decoding is transport/presentation decoding only; parsed values are emitted unchanged for Python to validate.
- A snapshot missing a reachable parameter's `draft_values[path]` renders a visible error instead of falling back to `selected_value` or `default_value`.
- Numeric controls emit a number when non-empty and the raw empty string when cleared, leaving acceptance/rejection to Python.
- Package tests use Node's built-in test runner, jsdom, and Testing Library; CI uses Node 22 and the repository's current checkout/setup-action major versions.

- `npm install --package-lock-only --ignore-scripts` updates the committed lockfile for the explicitly added test dependencies before the required clean `npm ci` verification.
- `rm -rf packages/react/dist` removed ignored build output before the final clean build so deleted legacy modules could not survive as stale package contents.
- The first bare `git commit` attempt was blocked because the repository hook could not find `pre-commit`; the retry runs `git commit` through `uv run` so the existing project hook executes in its declared environment.
